SPARQL on a plain element — data-from-query

Any element made query-driven: the sparql capability's activator reads data-from-query + the query attributes (endpoint, sparql/pattern, var-*) and renders the result based on the host element's tag — there is no view. A <ul> gets one <li> per row; a <select> gets one <option> per row (and fires sol-select); any other element renders nothing and the W3C SPARQL 1.1 Query Results JSON is left on el.swcData for you to use. (Want swc to pick a view instead? Use the <sol-query> element.)

The same query on two different elements — the tag decides the output:

A <ul> → one <li> per row

    A <select> → one <option> per row
