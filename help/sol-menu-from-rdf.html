Menus from RDF definition files

from-rdf is an opt-in capability: a page must load the menu-from-rdf add-on (this demo names it alongside sol-basic in the loader tag). To switch it on for the individual components, see Adding from-rdf to a menu.

Define a menu entirely in RDF using the ui: vocabulary.
Run it as a GUI with a simple HTML web component.
Run it in a terminal with a single command.

Given the same RDF menu definition as the GUI, the following command in a terminal :


   $ node sol-menu-cli.js data/demo-menu.ttl#MainMenu

Produces this interactive menu

This is the full HTML to load the live demo

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.0/component-interop.js"
        data-stage="local"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-basic sol-query sol-login menu-from-rdf"
></script>

<sol-menu from-rdf="data/demo-menu.ttl#MainMenu"></sol-menu>

The sol-menu web component can be used with a single from-rdf attribute. The value is a URI with a fragment identifier (e.g. menu.ttl#MainMenu) pointing to the root ui:Menu subject.

The selected menu item is shown in the menu's own panel, or — when the host page gives the <sol-menu> a region="#selector" (or a parent declares one) — in that region instead. 

ui:Menu
  ui:label       # (required) display name
  ui:parts       # (required) ordered rdf:Collection of ui:Link, ui:Component, and/or ui:Menu items
  ui:orientation # (optional) ui:Horizontal (default) or ui:Vertical

ui:Link
  ui:label       # (required) display name
  ui:href        # (optional) URL to load (same-origin → sol-include, external → iframe)
  ui:contents    # (optional) literal HTML rendered inline directly
  ui:icon        # (optional) icon URL

  note: a ui:Link must have either a ui:href or a ui:contents triple.

ui:Component
  ui:label       # (required) display name shown in the menu
  ui:name        # (required) EITHER a custom-element tag (hyphenated, e.g. "sol-query")
                 #            OR a hyphen-free command key (e.g. "installPod")
  ui:attribute   # (optional) schema:PropertyValue pairs — element attributes,
                 #            or, for a command, args carried in the event detail
                 # e.g. [ schema:name "sparql" ; schema:value "SELECT ..." ]
  ui:icon        # (optional) icon URL
  acl:mode       # (optional) access mode the item needs, e.g. acl:Write (WAC).
                 #            Rendered with part="requires-write"; the host app
                 #            decides the policy (hide / disable / …).

  note: a hyphenated ui:name names a custom element the app loads/defines.
  A hyphen-free ui:name is a command: clicking dispatches a bubbling
  sol-command (detail: { command, params }) for the host app to
  resolve against its own registry — no code is named in the RDF. (Menus and
  <sol-dropdown-button> dispatch on click; <sol-tabs> skips command items.)
