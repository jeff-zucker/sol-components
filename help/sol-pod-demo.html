sol-pod component

Attributes

  • source — pod storage URL or comma/space-separated list. Omit to auto-discover from the current origin / WebID.
  • login — CSS selector for an external <sol-login>. Omit to use the built-in one in the header.
  • login-mode, login-callback, issuers, side — forwarded to the built-in <sol-login>.
  • pod-click-action — function (assigned via JS property) or named handler. Receives (item, podEl) when the user clicks the per-item icon, presses Enter, or double-clicks. Item carries { url, name, displayName, isContainer, contentType }.
  • handler — alias for pod-click-action.
  • gear-icon — replaces the default ⚙ per-item button. URL (contains / or ends in svg/png/jpg/gif/webp) → renders as <img>; otherwise used as text.
  • pods-group — shared pod-list scope across multiple <sol-pod>s on the page. Absent = default shared group; 'none' = standalone registry.

Pod selector display

The pod dropdown strips the http:// / https:// prefix from each option for compactness — https://alice.solidcommunity.net/ shows as alice.solidcommunity.net/. The option's value stays the full URL so selection / fetch are unchanged.

The selector also has min-width: 0 in flex layouts so the login + settings buttons stay visible even in narrow sidebars; long URLs ellipsis-truncate inside the select.

Breadcrumb gear (current-container ops)

The breadcrumb row has a ⚙ pushed to the right edge that opens the <sol-pod-ops> modal scoped to whatever container the pod is currently showing — including the root / home view. Same modal you get for per-item clicks; just operates on "this folder" instead of a specific file.

Pod discovery

With no source attribute, sol-pod.discover() reads /.meta, /.well-known/solid, and /.acl at the current origin for owner WebIDs, then loads each WebID's profile and harvests pim:storage triples. It also walks owl:sameAs in both directions — a profile that asserts <remote> owl:sameAs <local> chains the same as one that asserts the inverse, so authoring direction doesn't matter. Logs each step at console.info('[pod-ops] …').

Discovery only fires when the shared pod registry is empty. A host that seeded the registry (e.g. podz restoring the previous session's pod list) is authoritative — sol-pod treats the seed as the dropdown's contents and skips discovery. Clear the seed to force re-discovery.
