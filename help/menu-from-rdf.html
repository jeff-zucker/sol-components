Adding from-rdf to a menu

By default <sol-menu>, <sol-tabs> and <sol-dropdown-button> are declarative-only: you author their items as light-DOM children (see the per-component help). To build the same menu from an RDF document instead, switch on one opt-in capability — the menu-from-rdf add-on — and point a from-rdf attribute at a ui:Menu subject. The add-on is the only piece that pulls in rdflib; without it, from-rdf is inert (the component logs a one-line notice and renders nothing from RDF).

1 · Switch the capability on

Load the add-on alongside the component. Pick the form that matches how you already load components:

<!-- The loader: name the component family + the add-on; rdflib rides in -->
<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.0/component-interop.js"
        data-stage="local"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-basic menu-from-rdf"
></script>

<!-- …or ESM: import the add-on once, anywhere -->
<script type="module">
  import 'sol-components/sol-menu.js';
  import 'sol-components/menu-from-rdf.js';   // ← activation
</script>

Importing menu-from-rdf once activates from-rdf for every member of the menu family on the page — sol-menu, sol-tabs and sol-dropdown-button alike — in any load order.

2 · Point from-rdf at a ui:Menu

The attribute is a URL with a fragment naming the root ui:Menu subject. The same RDF shape drives any member of the family — swap the tag, keep the document.

<sol-menu            from-rdf="menu.ttl#MainMenu"></sol-menu>
<sol-tabs            from-rdf="menu.ttl#MainMenu"></sol-tabs>
<sol-dropdown-button from-rdf="menu.ttl#MainMenu"></sol-dropdown-button>

3 · The ui:Menu shape

A ui:Menu has a ui:label and a ui:parts list of items. Each item is a ui:Link (a URL to load), a ui:Component (a custom element, or — when ui:name has no hyphen — a command), or a nested ui:Menu (a submenu).

@prefix ui:     <http://www.w3.org/ns/ui#> .
@prefix schema: <http://schema.org/> .

<#MainMenu> a ui:Menu ; ui:label "Demo" ;
  ui:parts ( <#New> <#Open> <#Tools> ) .

<#New>  a ui:Component ; ui:label "New"  ; ui:name "newItem" .   # command
<#Open> a ui:Component ; ui:label "Open" ; ui:name "openItem" .  # command

<#Tools> a ui:Menu ; ui:label "Tools" ;                          # submenu
  ui:parts ( <#Export> <#Settings> ) .

<#Export> a ui:Component ; ui:label "Export" ; ui:name "export" ;
  ui:attribute [ schema:name "format" ; schema:value "csv" ] .   # → params
<#Settings> a ui:Component ; ui:label "Settings" ; ui:name "settings" .
TermOnMeaning
ui:labelany itemDisplay text.
ui:partsui:Menu Ordered list (an RDF Collection) of the menu's items.
ui:orientationui:Menu ui:Horizontal (default) or ui:Vertical.
ui:href / ui:contentsui:Link A URL to load, or literal HTML to render inline.
ui:nameui:Component A hyphenated custom-element tag to mount, or a hyphen-free command key (click → sol-command).
ui:attributeui:Component schema:name/schema:value pairs — element attributes, or a command's params.
ui:iconlink / componentIcon URL.

The canonical shape lives in shapes/menu.shacl. A command's ui:name is just a key — no code is named in the RDF; your app maps the key to a handler when it receives the sol-command event.
