sol-login

A sign-in / sign-out button for Solid pods. Drop one on your page and it gives the user a way to log in — and once they have, the other sol-* components on the page can read and write that user's private data. You usually need just one per page.

Use it in your HTML

Add the loader once, near the top of your page. Listing sol-login in data-components is what turns on logging in:

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.1/component-interop.js"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-login"></script>

Then drop the tag wherever you want the sign-in button to appear:

<sol-login></sol-login>

Options

Some attributes here are shared across components — see Common attributes.

AttributeWhat it does
issuers A comma-separated list of sign-in providers to offer as quick-pick buttons (for example https://solidcommunity.net). Without it the user is asked to type one in.
mode How the sign-in happens. redirect (the default) takes the whole page to the provider and back. popup opens a small window instead, so your page never reloads and anything the user had on screen stays put.
popup-callback The address of the little page the popup returns to after signing in. Only needed with mode="popup".
visible Keep the button on screen at all times. Without it the component stays hidden and only appears when a sign-in is needed.
side A name for this login. Give two <sol-login> tags different side values (in popup mode) to be signed into two different pods at the same time — for example a left pod and a right pod.

Examples

<!-- the simplest version: asks the user which provider to use -->
<sol-login></sol-login>
<!-- offer ready-made sign-in buttons -->
<sol-login issuers="https://solidcommunity.net, https://login.inrupt.com"></sol-login>
<!-- sign in via a popup, so the page doesn't reload -->
<sol-login

    mode="popup"
    popup-callback="popup-auth-callback.html"
    issuers="https://solidcommunity.net,https://login.inrupt.com"

></sol-login>
<!-- two logins at once, one per pod -->
<sol-login mode="popup" side="left"  issuers="…"></sol-login>
<sol-login mode="popup" side="right" issuers="…"></sol-login>

Once a <sol-login> is on the page and the user signs in, the other data components automatically use that access — you don't have to wire anything up. If a component tries to read something private and the user isn't signed in yet, the login prompt appears on its own.
