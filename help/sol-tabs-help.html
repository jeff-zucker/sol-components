sol-tabs component

A tabbed container. Fill it with <a href> children — each anchor becomes a tab whose label is the link text and whose content is loaded from the href the first time you open it. The bar hides itself when there is only one tab.

Authoring

Each tab is an anchor. The href is the content URL; the link text is the tab label. By default the content is mounted with sol-include (fetch & show HTML/markdown/text); set handler="sol-*" on an anchor — or on <sol-tabs> as a default for every tab — to mount a different component instead. The href is handed to that component as both source and endpoint, and every other attribute on the anchor is forwarded too.

To keep a standard <a> valid you may write any forwarded attribute as data-*; the prefix is stripped when forwarding (data-handler picks the component, data-view → view, and so on).

Attributes

AttributePurpose
handler Default component tag for every tab (an anchor's own handler overrides it). Falls back to sol-include.
keep-alive Render every tab once into its own persistent pane and switch by toggling visibility — nothing is torn down, so scroll position, media playback and in-flight state survive a switch.
orientation horizontal (default) or vertical.
variant sub for the slimmer nested-subtab styling.

A non-anchor child (a button or custom control) is treated as a toolbar action and re-homed onto the tab bar instead of becoming a tab; mark an anchor slot="actions" to force it there too. Switching tabs fires sol-tab-change (detail: { name }).

Examples

<!-- three tabs, content fetched on first open -->
<sol-tabs>
  <a href="overview.html">Overview</a>
  <a href="notes.md">Notes</a>
  <a href="about.html">About</a>
</sol-tabs>

<!-- keep every panel alive (audio/scroll survive a switch) -->
<sol-tabs keep-alive>
  <a href="player.html">Player</a>
  <a href="queue.html">Queue</a>
</sol-tabs>

<!-- a default handler for all tabs + a forwarded attribute -->
<sol-tabs handler="sol-live-edit">
  <a href="readme.md">Readme</a>
  <a href="todo.md" data-view="preview">Todo</a>
</sol-tabs>

<!-- a toolbar action alongside the tabs (non-anchor child) -->
<sol-tabs>
  <a href="a.html">A</a>
  <sol-button inline source="help.html">?</sol-button>
</sol-tabs>
