PodOS + Solid Web Components — glueless via manifests

Two independent component libraries on one page, with no bridge script. sol-loader reads both libraries' interop manifest blocks and pairs swc's consume declarations with PodOS's provide declarations: when PodOS emits pod-os:loaded, the loader hands its os.store to SolidWebComponents.rdf.useStore and its authenticatedFetch to SolidWebComponents.adoptFetch. So sol-query reads the same graph and authenticates as the same user as the pos-* components — and PodOS's pod-os:resource-loaded drives sol-query's endpoint through the shared resource channel. Neither library is patched; swc gained only generic, library-neutral adoption hooks.

Solid Web Components · sol-* components

Editing a pos-* element with swc — no patch needed: give it a SHACL shape and sol-form auto-generates an editor. With the rdf capability loaded, add attributes to any element — shape="…shacl", subject="…uri" (for elements whose subject isn't in source), edit="inPlace|collected" — or declare it once in a manifest's interop.editable. This demo doesn't wire a live editor only because that needs a SHACL shape authored for the profile. Uses a public resource so it renders with no login; sign in via <pos-login> to exercise the shared authenticated path.
