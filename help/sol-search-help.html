sol-search

A search box that can use several search engines. The visitor types a query, picks an engine, and the results open in a reader window. You can show it as a small icon that pops open a panel, or as an always-visible strip.

Try it

Use it in your HTML

Add this line once, near the top of your page — it makes the component available:

<script src="dist/sol-loader.min.js" data-bundles="sol-search"></script>

Then drop the tag wherever you want the search box:

<sol-search default-engine="ddg"></sol-search>

Options

AttributeWhat it does
view button (the default) shows a small icon that opens a search panel when clicked — handy in a tight header. inline shows the input, Go button, and engine choices all the time — handy for a dedicated search strip.
engines A JSON list of engines, each { id, label, url }. The visitor's query is added to the end of url. Use this to set your own engine list right on the tag.
default-engine Which engine starts selected — give its id (or full web address). If left off, the first engine is used.
placeholder The greyed-out hint text in the input box (default Search…).

Some attributes here are shared across components — see Common attributes. In particular, source can point at a small settings file listing your engines instead of using the engines attribute.

If you give neither source nor engines, sol-search uses a built-in set: DuckDuckGo, Google, Wikipedia, prefix.cc, LOV, Etymology, YouTube, and Wayback.

Examples

An always-visible strip with your own engine list:

<sol-search view="inline" engines='[
  { "id":"ddg", "label":"DuckDuckGo", "url":"https://duckduckgo.com/?q=" },
  { "id":"g",   "label":"Google",     "url":"https://www.google.com/search?q=" }
]'></sol-search>

Or point source at a small settings file that lists the engines. In Turtle it looks like this:

@prefix ui: <http://www.w3.org/ns/ui#> .
@prefix bk: <http://www.w3.org/2002/01/bookmark#> .

<#SearchEngines> a bk:Topic ; ui:label "Search engines" .

:ddg a ui:Link ; ui:label "DuckDuckGo" ;
     bk:recalls  <https://duckduckgo.com/?q=> ;
     bk:hasTopic <#SearchEngines> .
:wp  a ui:Link ; ui:label "Wikipedia" ;
     bk:recalls  <https://en.wikipedia.org/w/index.php?search=> ;
     bk:hasTopic <#SearchEngines> .

Then point at it with the #Topic on the end:

<sol-search source="data/search-engines.ttl#SearchEngines"></sol-search>
