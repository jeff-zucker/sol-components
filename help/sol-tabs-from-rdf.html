Tabs from RDF definition files

Define a tab set entirely in RDF using the ui: vocabulary.
It is the same ui:Menu shape <sol-menu> consumes — one document can drive either element.
A nested ui:Menu becomes a tab whose body is a slimmer sub-tab strip.

This is the full HTML to load the live demo

<script src="../dist/vendor/@comunica-query-sparql.umd.js"></script>
<script src="../dist/solid-web-components.bundle.min.js" type="module"></script>

<sol-tabs from-rdf="../data/demo-tabs.ttl#MainTabs"></sol-tabs>

The sol-tabs web component can be used with a single from-rdf attribute. The value is a URI with a fragment identifier (e.g. tabs.ttl#MainTabs) pointing to the root ui:Menu subject. With no fragment, the first ui:Menu-typed subject in the document is used.

Each ui:Link or ui:Component part becomes a tab. A nested ui:Menu part becomes a tab whose content is a <sol-tabs variant="sub"> strip of that group's own children. 

ui:Menu
  ui:label       # (required) display name
  ui:parts       # (required) ordered rdf:Collection of ui:Link, ui:Component, and/or ui:Menu items
  ui:orientation # (optional) "horizontal" (default) or "vertical"
  ui:linkTarget  # (optional) CSS selector; content renders there instead of the built-in panel

ui:Link
  ui:label       # (required) display name
  ui:href        # (optional) URL to load (via handler component or default sol-include)
  ui:contents    # (optional) literal HTML rendered inline directly
  ui:handler     # (optional) a ui:Component used to render the href
  ui:icon        # (optional) icon URL — shown by sol-menu; sol-tabs uses the label text

  note: a ui:Link must have either a ui:href or a ui:contents triple.

ui:Component
  ui:label       # (required) display name shown on the tab
  ui:name        # (required) custom element tag, e.g. "sol-query"
  ui:attribute   # (optional) schema:PropertyValue pairs passed as element attributes
                 # e.g. [ schema:name "sparql" ; schema:value "SELECT ..." ]
  ui:icon        # (optional) icon URL

  note: a nested ui:Menu inside ui:parts renders as a tab containing a
  slimmer <sol-tabs variant="sub"> strip of its children.
