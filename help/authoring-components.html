Add your component to the ecosystem

You wrote a web component. This page covers the sol-components-specific part of being a good citizen: letting swc capabilities (like editing) enhance your component, and offering an enhancement capability of your own. Each step is optional and independent.

You never import any sol-components source code — everything is plain custom elements and DOM events.

Loading and resource-sharing are a separate, library-agnostic concern. To let people load your component with one tag (no URLs, no build), and to share a store, auth, or other resources between libraries, you write a component-interop manifest — documented with the broker, not here: component-interop (the manifest format, provides/consumes, and the rule that exposing a value is data; adopting one is behavior). This page is only the swc capability hooks.

Step 1 — Make it a custom element

Give it a hyphenated tag with your own prefix (so it never collides with anyone else's), and register it:

class AcmeMap extends HTMLElement { /* … */ }
customElements.define('acme-map', AcmeMap);

That's all it takes to plug in — a custom element is a first-class citizen of the page.

Step 2 — (Optional) Let capabilities enhance your component

Declare the extension points you offer with one static getter. The handy one is edit: point it at a small file listing your fields and the page's editing capability builds your component an edit form and a ✏️ pencil, for free:

class AcmeMap extends HTMLElement {
  static get extensionPoints() {
    return { edit: { shape: 'shapes/map-fields.ttl' } };
  }
  get source() { return this.getAttribute('source'); }   // where it reads config
  reload() { /* re-read source, re-render */ }            // called after a save
}

That's the whole opt-in — no glue. The same idea is open-ended: a capability can define other points (e.g. annotate), and any component listing that point in extensionPoints gets enhanced.

Step 3 — (Optional) Provide a capability that enhances others

To offer a capability — one that enhances other people's components — write a module that finds and enhances every component offering your extension point, now or whenever one mounts later:

import { observeExtensionPoint } from 'sol-components/core/extension-points.js';

observeExtensionPoint('annotate', (el, spec) => {
  // attach your enhancement to `el` using `spec`
});

You never import the components you enhance; they never import you — you meet in the DOM. Have the module load when its trigger attribute appears by adding an attributes entry to your manifest (see the component-interop docs); there is no data-extend-with — capabilities auto-load by presence.
