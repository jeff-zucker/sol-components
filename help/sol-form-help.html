sol-form

Turns a description of some data into a ready-made edit form. You give it a small file listing the fields you want (their names and types) and the data file to edit, and it builds the inputs, fills them in, and saves changes back for you. Use it whenever you need a form over a Solid data file without hand-writing the HTML for every field.

Try it

shapes/weather-settings.shacl read-only
Generated form — sol-weather settings no-edit

On the left is the field description (a SHACL file). On the right is the form sol-form built from it. This demo is read-only.

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="dist/sol-loader.min.js" data-with="rdf"></script>

Then point a form at a field description and the data file to edit:

<sol-form
    shape="shapes/weather-settings.shacl"
    subject="data/weather-settings.ttl#Settings"
></sol-form>

That reads the data, shows one input per field, and saves your edits back to the data file automatically.

Options

AttributeWhat it does
subject The data being edited — the data file, with an optional #Name on the end to pick one entry inside it (e.g. data/weather.ttl#Settings). The form reads from here and, unless you set save-to, writes back here too.
no-edit Show the form read-only: the current values are visible but can't be changed and there's no save bar. Handy for a "look, don't touch" view of a configuration.

Some attributes here are shared across components — see Common attributes. In particular shape (the field description) and save-to (an alternate place to write edits) are covered there.

Examples

<!-- a normal editable form built from a field description -->
<sol-form shape="shapes/profile.shacl" subject="profile/card.ttl#me"></sol-form>

<!-- read-only: show the values without letting anyone change them -->
<sol-form shape="shapes/profile.shacl" subject="profile/card.ttl#me" no-edit></sol-form>

<!-- read from one file, save edits to another -->
<sol-form
    shape="shapes/profile.shacl"
    subject="templates/profile.ttl#me"
    save-to="https://me.example/profile/card.ttl#me"
></sol-form>

The field description (the shape file) decides which inputs appear and what kind each is — a text box, a number, a checkbox, a dropdown, and so on. Add a field there and it shows up in the form automatically.
