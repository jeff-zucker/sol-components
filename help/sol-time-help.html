sol-time component

A compact clock widget. Always shows local and GMT; optionally adds a third timezone (label + UTC offset). Ticks once a minute — the seconds aren't displayed so finer updates would be busywork.

<script src="dist/sol-full.umd.min.js"></script>

<sol-time
  time-label="tokyo"
  time-offset="9"
></sol-time>
AttributeRequiredDescription
time-label optional Short label for an extra timezone reading (e.g. tokyo, mumbai).
time-offset required if time-label is set The extra timezone's offset from UTC in hours. Accepts fractional values: 5.5, -3.5.
source optional file.ttl#Subject — a schema:PropertyValue config block (see RDF config). HTML attributes still win over values from the TTL.

Examples

<!-- Local + GMT only -->
<sol-time></sol-time>

<!-- Local + GMT + a third timezone -->
<sol-time time-label="tokyo" time-offset="9"></sol-time>

<!-- Pull settings from a TTL file -->
<sol-time source="data/time.ttl#Settings"></sol-time>

Theming via CSS parts

Each label / value / separator is exposed as a shadow part: local-label, local-time, utc-label, utc-time, extra-label, extra-time, sep. Style them from the host page:

sol-time::part(local-time) { font-variant-numeric: tabular-nums; }
sol-time::part(extra-label) { color: var(--accent); }

Settings from RDF

Point source at a Turtle file with a schema:PropertyValue block. Setting names map to the HTML attributes:

PropertyValue nameHTML attribute
"timezone" time-label
"timezone-offset" time-offset
 
<sol-time source="data/time.ttl#Settings"></sol-time>

And data/time.ttl:

@prefix schema: <http://schema.org/> .

<#Settings>
  schema:additionalProperty <#timezone>, <#timezone-offset> .

<#timezone>
  a schema:PropertyValue ;
  schema:name "timezone"@en ;
  schema:value "tokyo" .

<#timezone-offset>
  a schema:PropertyValue ;
  schema:name "timezone-offset"@en ;
  schema:value 9 .

The PropertyValue pattern uses only standard schema.org predicates (PropertyValue, name, value, additionalProperty) — no invented vocabulary. The schema.org namespace is always http://schema.org/, not the HTTPS form.

This path needs rdflib on the page (it publishes window.$rdf).

Accessibility

  • The clock is a single inline element; no interactive controls.
  • All colours come from shared theme tokens (--text, --text-muted) so the widget follows the page's light / dark setting.
  • Numbers use font-variant-numeric: tabular-nums so each minute-tick keeps the digits aligned (no horizontal shimmer).
  • Each label / value can be styled or hidden via shadow parts (see the Attributes tab).
