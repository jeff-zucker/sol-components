sol-menu

A navigation menu with a built-in content panel. You fill it with links; clicking a link loads that page into the panel right beside the menu. It's a quick way to build a sidebar or nav bar where the content appears in place. The first link opens automatically, so the panel is never empty.

Try it

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.2/component-interop.js"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-basic"></script>

Then list ordinary links inside the tag. The href is the page to load and the link text becomes the menu label:

<sol-menu>
  <a href="home.html">Home</a>
  <a href="about.html">About</a>
</sol-menu>

Group links under a collapsible heading by wrapping them in a <submenu> with a <label>; you can nest these to any depth.

Options

AttributeWhat it does
icon (on a link) Show an icon instead of text (an image URL or inline SVG); the link text becomes its tooltip.

Several attributes here are shared across components — handler (which component opens a link, defaults to showing the page), orientation (sidebar or top bar), and if-logged-in (mark a link as needing sign-in). See Common attributes.

Examples

<!-- sidebar with a nested group -->
<sol-menu>
  <a href="home.html">Home</a>
  <submenu>
    <label>Docs</label>
    <a href="quickstart.md">Quickstart</a>
    <submenu>
      <label>API</label>
      <a href="api/query.md">Query</a>
      <a href="api/modal.md">Modal</a>
    </submenu>
  </submenu>
</sol-menu>

<!-- a top nav bar -->
<sol-menu orientation="horizontal">
  <a href="readme.md">Readme</a>
  <a href="changelog.md">Changelog</a>
</sol-menu>
