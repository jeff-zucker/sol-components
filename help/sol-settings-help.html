sol-settings

A one-tag settings page. Drop <sol-settings> on a page and it looks at the other components already there, finds the ones that can be edited, and builds a settings panel for each — one expandable section per component. You don't configure anything; it figures out the panels from what's on the page.

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.1/component-interop.js"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="rdf-bundle"></script>

Then put your components on the page and add one settings tag after them:

<sol-weather place="Portland, OR"></sol-weather>
<sol-menu from-rdf="data/menu.ttl#MainMenu" region="#main"></sol-menu>

<sol-settings></sol-settings>

Here it finds the weather card and the menu and gives you a panel to edit each. sol-settings takes no attributes of its own.

How it picks panels

A component shows up in settings if it knows how to edit itself — most of the editable ones (weather, clock, menu, search, and so on) already do. Each panel opens the component's own editor when you expand it, and saving there updates the live component on the page right away. A component that has no editor simply isn't listed.

Examples

<!-- a page chrome with several editable widgets and one settings panel -->
<sol-menu from-rdf="data/menu.ttl#MainMenu" region="#main"></sol-menu>
<sol-weather place="Portland, OR"></sol-weather>
<sol-time></sol-time>

<sol-settings></sol-settings>

One <sol-settings> gives you a section to edit the menu, the weather card, and the clock — no extra wiring per component.
