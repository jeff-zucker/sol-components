sol-dropdown-button

A small trigger button that drops a menu of actions when you click it — handy for an "⋮ More" or "Actions" button. You list the choices inside it; the menu opens on click and closes when you pick one, click away, or press Escape.

Try it

Last command: —

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="dist/sol-loader.min.js" data-bundles="sol-basic"></script>

Then write the button with a <menu> of choices inside it. Each choice is a <button> (an action your page reacts to) or an <a href> (opens a page or file):

<sol-dropdown-button label="⋮ Actions">
  <menu>
    <button handler="rename">Rename</button>
    <a href="about.html" region="modal">About</a>
  </menu>
</sol-dropdown-button>

Options

AttributeWhat it does
label The text on the trigger button (defaults to ⋮).
handler (on a choice) A plain word (like rename) makes the choice an action: clicking it tells your page which command was picked, so your own code can do the work. A component tag instead opens that component.
params (on an action choice) Extra information to pass along with an action, written as JSON — for example params='{"format":"csv"}'.
requires-write (on a choice) Marks a choice as owner-only, so the page can hide or disable it for visitors who aren't signed in.

Some attributes here are shared across components — see Common attributes.

Examples

<!-- a menu of actions your page handles -->
<sol-dropdown-button label="⋮ More">
  <menu>
    <button handler="install" requires-write>Install on my Pod…</button>
    <button handler="export" params='{"format":"csv"}'>Export</button>
    <a href="about.html" region="modal">About</a>
  </menu>
</sol-dropdown-button>

When someone picks an action, your page finds out which one and can respond:

<script>
  document.addEventListener('sol-command', (e) => {
    const { command, params } = e.detail;
    // do something based on which command was chosen…
  });
</script>
