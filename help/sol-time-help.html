sol-time component

A compact clock. Always shows local and GMT; optionally adds a third clock for a named timezone. Ticks once a minute (no seconds).

Live demo

Configure from RDF

Point source at a Turtle settings file. The third clock is named by one direct predicate — the IANA name supplies both the label and the UTC offset (computed at runtime), so there is no separate offset setting.

<sol-time source="data/time-settings.ttl#Settings"></sol-time>

data/time-settings.ttl:

Settings

PredicateTypeMeaning
schema:timezonestring An IANA timezone name, e.g. "Asia/Kolkata" or "America/New_York". The label shown is the last path segment (Kolkata); the offset is derived from the zone.

Reading the settings file needs rdflib on the page — the loader pulls it in transitively (deduped to one shared instance via the injected importmap). The schema.org namespace is always http://schema.org/, never the HTTPS form.

Theming

Each label / value / separator is a shadow part: local-label, local-time, utc-label, utc-time, extra-label, extra-time, sep.

sol-time::part(local-time)  { font-variant-numeric: tabular-nums; }
sol-time::part(extra-label) { color: var(--accent); }
