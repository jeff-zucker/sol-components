sol-button component

A declarative launcher: a trigger button that, on click, renders content into a region. What renders is the handler (a component, defaulting to sol-include); where it lands is region — a CSS selector (a pane on the page) or a keyword that conjures an ephemeral surface (modal / floating / tab / window) with no host element to author.

Attributes

AttributePurpose
handler Tag of the component to mount. Optional — when omitted it's inferred from source: same-origin → trusted sol-include, external URL → iframe.
region Where the result surfaces. A CSS selector → render into that pane (a keep-alive tab). Or a keyword — modal, floating, tab, window — conjured on click, no host element needed. Resolved by cascade: this attribute → a parent [region] container → an enclosing <sol-menu> → <sol-default region>.
source Forwarded to the handler (e.g. sol-include's source).
name Wrapper id (data-menu-item) for the pane tab; defaults to the element's id, else a slug of source.
replace Rebuild a pane wrapper's contents on every click instead of reusing (keep-alive) them.

Every other attribute is forwarded to the handler element, so authoring a launcher is like inlining the handler. The trigger is exposed as ::part(trigger). On activation it fires sol-button-activate (detail: { name, handler, wrapper }).

Examples

<!-- same-origin page into a pane (trusted sol-include) -->
<sol-button source="pages/settings.html" region="#main">Settings</sol-button>

<!-- pop a component in a modal, no host element -->
<sol-button handler="sol-query" source="data/reports.ttl" region="modal">Reports</sol-button>

<!-- external site → iframe inferred -->
<sol-button source="https://solidproject.org" region="#main">Website</sol-button>

<!-- a group of buttons sharing one region, declared once -->
<div data-region="#aux">
  <sol-button source="a.html">A</sol-button>
  <sol-button source="b.html">B</sol-button>
</div>

A pane region is an element the page declares (<main id="main">); the keyword surfaces (modal/floating) are the <sol-modal> / <sol-window> hosts, conjured per click. Style a pane's contents directly with ordinary CSS — the mounted result is light DOM.
