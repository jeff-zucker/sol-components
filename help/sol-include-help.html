sol-include

Fetches another text file — an HTML page, a markdown file, a script — and drops its contents straight into your page. It's the easy way to reuse a shared header, footer, or article without copying it everywhere. Markdown is turned into HTML for you, and you can pull in just one part of a page if you want.

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.0/component-interop.js"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-basic"></script>

Then point the tag at the file you want to pull in:

<sol-include source="https://example.org/article.html"></sol-include>

Options

AttributeWhat it does
source The file to fetch and show.
selector Pull in only the part of the file that matches this CSS selector.
raw Show the file's text exactly as written, instead of rendering it.
trusted Let the fetched HTML run as real HTML (and let any components inside it activate). Use only for files you trust — without it, HTML is shown as plain text for safety.

These attributes are shared across components — see Common attributes.

Examples

<!-- include a whole HTML page -->
<sol-include source="header.html"></sol-include>

<!-- include a markdown file (rendered as HTML) -->
<sol-include source="readme.md"></sol-include>

Pull in just one piece of a page with selector:

<sol-include
  source="https://example.org/blog.html"
  selector="article:first-of-type"
></sol-include>

Show a file's literal text with raw:

<sol-include source="script.js" raw></sol-include>

Build a page from reusable parts by nesting includes. A trusted page can itself contain sol-includes and other components, and they activate when pulled in:

<sol-include source="header.html" trusted></sol-include>
<sol-include source="body.html" trusted></sol-include>
<sol-include source="footer.html" trusted></sol-include>
