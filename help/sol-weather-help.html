sol-weather component

A current-conditions card — icon, place, temperature, and max-precipitation chance for the next N hours. Data from the free, key-less Open-Meteo API (sent with Access-Control-Allow-Origin: *, so no proxy). Refreshes every 10 minutes.

Live demo

Configure from RDF

Point source at a Turtle settings file. All settings use direct predicates on the topic the file's foaf:primaryTopic names.

<sol-weather source="data/weather-settings.ttl#Settings"></sol-weather>

data/weather-settings.ttl:

Settings

PredicateTypeMeaning
geo:latdecimalLatitude (−90 to 90). Required.
geo:longdecimalLongitude (−180 to 180). Required.
schema:addressLocalitystringPlace name shown on the card.
ui:temperatureUnitIRIui:Fahrenheit | ui:Celsius | ui:Both.
time:hoursintegerMax-precip lookahead, in hours (default 12).

Reading the settings file needs rdflib on the page (it publishes window.$rdf). The schema.org namespace is always http://schema.org/, never the HTTPS form.

Theming

The card and its cells are shadow parts: card, icon, place, temp, desc, rain. Colours come from the shared theme tokens in web/styles/root.css.

sol-weather::part(temp) { font-size: 1.4em; font-weight: 600; }
sol-weather::part(desc) { display: inline; } /* show verbose description */
