sol-weather component

A compact current-conditions card: weather icon · place · temperature · max-precip probability for the next N hours. Backed by the public Open-Meteo API — no key required. Pass lat + lon directly or let the component geocode a place name. Refreshes every ten minutes; aborts in-flight fetches on disconnect.

<script src="dist/sol-full.umd.min.js"></script>

<sol-weather
  place="Portland, OR"
  units="both"
></sol-weather>
AttributeRequiredDescription
lat required (or place) Decimal latitude (e.g. 45.52).
lon required (or place) Decimal longitude (e.g. -122.68).
place required (or lat + lon) Free-text place name. Geocoded once via Open-Meteo; also used as the card's label.
units optional metric | imperial | both (default both).
hours-window optional Max-precip lookahead in hours (default 12).
source optional file.ttl#Subject — a schema:PropertyValue config block (see RDF config). HTML attributes still win over values from the TTL.

Examples

<!-- Geocode a place name -->
<sol-weather place="Portland, OR"></sol-weather>

<!-- Pin coordinates (no geocode round-trip) -->
<sol-weather lat="45.52" lon="-122.68" units="imperial"></sol-weather>

<!-- Pull everything from a TTL config -->
<sol-weather source="data/weather.ttl#Settings"></sol-weather>

Theming via CSS parts

The card and each cell are exposed as shadow parts: card, icon, place, temp, desc, rain.

sol-weather::part(card)  { padding: 1rem; }
sol-weather::part(temp)  { font-size: 1.4em; font-weight: 600; }
sol-weather::part(desc)  { display: inline; } /* show the verbose
                                                 description, hidden
                                                 by default */

Settings from RDF

Point source at a Turtle file with a schema:PropertyValue block. Setting names map to the HTML attributes:

PropertyValue nameHTML attribute
"latitude" lat
"longitude" lon
"place" place
"units" units
"forecast-window" hours-window
 
<sol-weather source="data/weather.ttl#Settings"></sol-weather>

And data/weather.ttl:

@prefix schema: <http://schema.org/> .

<#Settings>
  schema:additionalProperty
    <#latitude>, <#longitude>, <#place>,
    <#units>, <#forecast-window> .

<#latitude>        a schema:PropertyValue ; schema:name "latitude"@en        ; schema:value 45.52 .
<#longitude>       a schema:PropertyValue ; schema:name "longitude"@en       ; schema:value -122.68 .
<#place>           a schema:PropertyValue ; schema:name "place"@en           ; schema:value "Portland, OR" .
<#units>           a schema:PropertyValue ; schema:name "units"@en           ; schema:value "both" .
<#forecast-window> a schema:PropertyValue ; schema:name "forecast-window"@en ; schema:value 12 .

This path needs rdflib on the page (it publishes window.$rdf). The schema.org namespace is always http://schema.org/, not the HTTPS form.

Open-Meteo

Forecast data and geocoding both come from Open-Meteo, which is free and requires no API key. The component hits two endpoints:

  • https://geocoding-api.open-meteo.com/v1/search?name=<q>&count=1 — only when place is set and lat/lon aren't.
  • https://api.open-meteo.com/v1/forecast?latitude=…&longitude=…&current_weather=true&hourly=…&timezone=auto — the actual weather, refreshed every 10 minutes.

Hourly fields requested: temperature_2m, precipitation_probability, weathercode. Weather codes are mapped to a description + emoji from the standard WMO 4677 set (clear sky, partly cloudy, rain, snow, thunderstorm, etc.).

Offline

When navigator.onLine is false, the card hides itself rather than displaying stale numbers as if they were current. The next online refresh re-populates it.

CORS

Both Open-Meteo endpoints send Access-Control-Allow-Origin: *, so no proxy is needed — sol-weather fetches them directly from the browser.

Accessibility

  • The card is a labelled live region (role="region" + aria-live="polite"), so screen readers announce conditions when they change rather than re-reading on every refresh.
  • Errors are announced via a separate role="alert" strip that stays hidden until a fetch fails.
  • Temperature values use font-variant-numeric: tabular-nums so a refresh doesn't horizontally shift the surrounding text.
  • All colours come from shared theme tokens, so the widget follows the page's light / dark setting.
