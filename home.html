solid-web-components

Access Solid and Linked Data resources using HTML custom elements

Invoking components Theming components Creating custom views
sol-tabs Tabbed container — fill it with <a href> children and each becomes a tab whose content loads on first open. Keep-alive panes preserve scroll, media playback and form state across switches.
sol-menu Sidebar (or top) navigation plus a content panel. Anchor children become entries; nest <submenu> groups to any depth. Clicking an entry loads it into the panel.
sol-dropdown-button A trigger button that drops a menu in a floating popup. Declare items inline in a <menu>; a bare-name handler fires a sol-command, a tag or <a href> mounts content.
sol-button A declarative launcher: a trigger that renders a handler component into a region — a page pane or a conjured modal / floating / window surface.
menus from RDF Drive sol-menu, sol-tabs or sol-dropdown-button from a ui:Menu RDF document instead of HTML children: switch on the opt-in menu-from-rdf add-on and set a from-rdf attribute. (Showcase: a menu in GUI & terminal.)
sol-query Search data sources using Comunica with inline or stored SPARQL or using triple patterns. Display the results as a table, an auto-complete field, a rolodex, other built-in and custom views.
sol-login Dynamically generated login/logout button. Once logged in, the authenticated fetch is propagated to all other fetches, including Comunica queries and HTML/Markdown page includes.
sol-include Fetch text based documents and display their contents inline. Supports authenticated fetch, robust modular page construction, CSS/RDFa filetering, Markdown conversion, and always sanitizes by default.
sol-pod Opens an expand/collapse tree of one or more pods, their containers, and resources. You can control what happens when parts of the tree are clicked so it becomes a generic resource picker.
sol-feed Display RSS and Atom feeds three ways: a single feed, a multi-source picker, or a Google-News-like grid of randomized article cards with images and reveal-on-focus descriptions.
sol-calendar Inline agenda view of any public iCalendar (ICS) feed — Google, Apple, Outlook, Proton, Solid pod. Merges multiple calendars into one sorted list; meeting URLs (Jitsi / Zoom / Meet) become click targets on the event titles.
sol-search Multi-engine search field with an icon-trigger popup or an always-visible inline layout. Engines come from a bookmark-style RDF list, a JSON attribute, or a built-in default list (DuckDuckGo, Google, Wikipedia, prefix.cc, LOV, …).
sol-weather Compact current-conditions card backed by the no-key-required Open-Meteo API. Pass coordinates or let it geocode a place name; choose metric, imperial, or both; tune the max-precip lookahead window.
sol-time Compact clock widget — always shows local and GMT; optionally adds a third timezone (label + UTC offset). Ticks once a minute; settings can come inline or from a small RDF config block.
sol-form Auto-saving editable forms bound to RDF. Two paths: shape-driven — point at a SHACL shape + data document and sol-form generates the inputs from the shape; form-driven — supply a hand-authored ui:Form TTL and solid-ui renders it. Includes a live SHACL-to-form demo on the help page.
sol-solidos Mount the SolidOS data browser (mashlib panes) inside a custom element; set source to navigate. Requires loading mashlib separately as a UMD — not in the all-in-one bundle. Help page covers the standard wiring, the known empty-window.Mashlib quirk, and a panes-only embedding pattern for hosts (like an iframe) that just want the outline.
dashboard A worked-example page combining sol-weather, sol-time, sol-calendar, and sol-feed into a single dashboard layout. Open dashboard.html in the repo root to see them all working together.
