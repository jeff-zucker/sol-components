sol-calendar component

An inline agenda. Point it at one or more public iCalendar (ICS) feeds — Google, Apple, Outlook, Proton, or a Solid pod — and it renders upcoming events as a merged, sorted list.

Live demo

Configure from RDF

Point source at a Turtle file and select a settings record with a #fragment. Each record carries its feed list and display options as direct predicates — one or more dct:source IRIs give the ICS feeds; multiple values in one record are fetched in parallel and merged into a single agenda.

<sol-calendar source="data/calendar-settings.ttl#All"></sol-calendar>

data/calendar-settings.ttl:

Settings

PredicateTypeMeaning
dct:sourceIRI (repeatable) ICS feed URL. Repeat for a merged multi-source agenda.
dct:formatstring "ics" or "google" — selects the parser.
ui:viewstring "agenda" (v1) · month / week / day planned.
time:daysinteger Lookahead window, in days.
schema:numberOfItemsinteger Optional cap on rendered events.

An HTML attribute of the same name always overrides the value from the TTL. Reading the file needs rdflib on the page (it publishes window.$rdf). The schema.org namespace is always http://schema.org/, never the HTTPS form.

proxy

Most providers don't serve their ICS endpoints with permissive CORS, so the browser can't fetch them directly. proxy is not a settings-file predicate — it is a shared default read from the <sol-default> singleton, or set as an HTML attribute on the element. If the pattern contains the token {url} it is substituted; otherwise the encoded URL is appended.

<sol-calendar source="data/calendar-settings.ttl#All"
              proxy="http://localhost:3002/proxy?uri="></sol-calendar>

Omit proxy only for feeds that already send Access-Control-Allow-Origin (a Solid pod, a self-hosted ICS file).

Finding the ICS feed URL

dct:source must be an iCalendar export URL (returns text/calendar, starts with BEGIN:VCALENDAR) — not the URL of the calendar page in your browser, which returns HTML and fails with "non-ICS body". curl <url> should print BEGIN:VCALENDAR.

  • Google — Settings and sharing → "Integrate calendar". The secret address in iCal format works for any calendar (the token is the credential — treat it like a password). The public address works once the calendar is made public.
  • Apple iCloud — right-click the calendar → Share → Public Calendar; swap webcal:// for https://.
  • Outlook — Settings → Calendar → Shared calendars → Publish a calendar; copy the ICS link.
  • Proton — calendar settings → Share publicly → "Subscribe with calendar app".
  • Self-hosted / Solid pod — any URL returning text/calendar.

Meeting links

When a video-meeting URL is found it becomes the event title's href. Lookup order: the ICS URL: property; a LOCATION: that is wholly a URL; the first known meeting host in DESCRIPTION: (Jitsi, Zoom, Google Meet, Teams, Whereby); then any URL in the description.
