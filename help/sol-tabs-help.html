sol-tabs

A tabbed box. You fill it with links; each link becomes a tab, and clicking a tab loads that page into the panel below. The content for a tab is fetched the first time you open it. If there's only one tab, the bar hides itself.

Use it in your HTML

Add this line once, near the top of your page — it makes the components available:

<script src="https://cdn.jsdelivr.net/npm/component-interop@0.2.1/component-interop.js"
        data-manifest="../dist/sol-components.manifest.json"
        data-components="sol-basic"></script>

Then list a link for each tab. The href is the page to load and the link text becomes the tab label:

<sol-tabs>
  <a href="overview.html">Overview</a>
  <a href="about.html">About</a>
</sol-tabs>

Options

AttributeWhat it does
keep-alive Keep every tab's content loaded in the background instead of rebuilding it each time. Switching tabs then preserves scroll position, playing media, and anything else in progress.
variant Set to sub for the slimmer styling used by tabs nested inside other tabs.

A couple of attributes here are shared across components — handler (which component opens a tab, defaults to showing the page) and orientation (horizontal or vertical). See Common attributes.

Examples

<!-- three tabs, content fetched on first open -->
<sol-tabs>
  <a href="overview.html">Overview</a>
  <a href="notes.md">Notes</a>
  <a href="about.html">About</a>
</sol-tabs>

<!-- keep every panel alive (audio/scroll survive a switch) -->
<sol-tabs keep-alive>
  <a href="player.html">Player</a>
  <a href="queue.html">Queue</a>
</sol-tabs>

A non-link child (such as a button) is placed on the tab bar as a toolbar control rather than becoming a tab:

<sol-tabs>
  <a href="a.html">A</a>
  <sol-button inline source="help.html">?</sol-button>
</sol-tabs>
