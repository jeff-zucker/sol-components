Sharing data, login & settings

Newly implemented. This surface ships now, but hasn't been exercised end-to-end in a real browser yet — treat runtime details as provisional and report anything that misbehaves.

A page can hold components from several different authors at once. They all share one data store, one login, and one set of settings — so your component and everyone else's read and write the same data and use the same signed-in user. You reach all of it through a single object the loader puts on the page, window.SolidWebComponents, with no imports.

Step 1 — get the object and wait until it's ready

The loader sets up window.SolidWebComponents and resolves .ready once components have loaded:

const SWC = window.SolidWebComponents;
await SWC.ready;

If you need one specific resource that might load after you, wait for just that one:

const rdf = await SWC.services.whenReady('rdf');

Step 2 — the shared data store

One RDF store for the whole page. Anything you load into it is visible to every other component, and deduped so it's only fetched once:

await SWC.rdf.load('https://my.pod/things.ttl');   // shared + deduped
const store = SWC.rdf.store;                        // the one rdflib store

Step 3 — the signed-in fetch

If there's a <sol-login> on the page and the user is signed in, SWC.fetch automatically carries their credentials; otherwise it's the normal fetch. You don't wire anything up:

const res = await SWC.fetch('https://my.pod/private.ttl');

React to the user signing in or out:

SWC.on(SWC.EVENTS.LOGIN,  () => this.reload());
SWC.on(SWC.EVENTS.LOGOUT, () => this.reload());

Step 4 — shared settings

Page-wide settings (for example a CORS proxy) live on one <sol-default>. Read them, and react when they change, instead of asking the user to configure each component separately:

const proxy = SWC.defaults.get('proxy');
SWC.defaults.onChange((name) => { if (name === 'proxy') this.reload(); });

Step 5 — check what's there, and degrade gracefully

Don't assume a resource exists — ask, so your component still works on a page that doesn't have it:

if (SWC.has('auth')) this.showSaveButton();   // only if login is available

Sending & receiving events

Components coordinate by emitting and listening for events on the document. Use the names in SWC.EVENTS rather than hardcoding strings, and fire your own with SWC.emit:

const off = SWC.on('acme-map-select', (e) => { /* … */ });   // listen
SWC.emit('acme-map-select', { id });                          // send
off();                                                        // stop listening

Your own component's events should be named your-tag-action and fire with bubbles: true, composed: true so anything on the page can hear them.
