Loading the components

The loader is one <script> tag you add near the top of your page. It makes the components available, so you can then use tags like <sol-button> or <sol-menu> anywhere in your HTML.

The loader tag

Add it once, in the <head> of your page:

<script src="dist/sol-loader.min.js"
        data-stage="local"
        data-bundles="sol-basic sol-feed"
        data-extend-with="auth"
></script>

It has three knobs:

  • data-bundles — the components (or families) you want; each token is a tag name, like sol-basic for the everyday UI plus widgets such as sol-feed.
  • data-extend-with — extra capabilities to switch on: auth (sign-in), sparql (queries), and rdf (the editing components).
  • data-stage — where the code is loaded from: local for files you host yourself, or cdn for loading over the internet.

Run your code when it's ready

If you have JavaScript that uses the components, wait for them to be ready first:

<script>
  await SolidWebComponents.ready;
  // your code here
</script>

If you'd rather listen for an event, the loader fires swc:ready on the document when everything is loaded.

Module / capability reference

A quick guide to which tokens to use for what you want to build.

You want…Loader tokensIt pulls in for you
The everyday UI (button, dropdown-button, include, menu, tabs, accordion, rolodex, modal, window, default)data-bundles="sol-basic"the basic UI components
Drive the menu/tabs family from a data filedata-bundles="sol-basic menu-from-rdf"+ data support
A widget (time, weather, search, calendar, feed, gallery)data-bundles="sol-feed" (etc.)that widget and what it needs
Pods (pod, live-edit; pod-ops + wac as sol-pod-extras)data-bundles="sol-pod sol-pod-extras sol-live-edit"the pod components
Sign-indata-extend-with="auth"sol-login
SPARQL queriesdata-extend-with="sparql"sol-query
The editing components (sol-tree-edit, sol-form, sol-settings)data-extend-with="rdf"the editing components

Add a <sol-login> to the page and, once the user is signed in, the data components automatically use that sign-in — no wiring needed.
