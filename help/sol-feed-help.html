sol-feed component

Fetches and displays RSS or Atom feeds. One element, three layouts — pick a layout with the view attribute: a single feed, a topic picker scoped to one topic's subtree, or an all grid that merges and randomizes articles across every source in a topic tree.

<script src="dist/sol-feed.umd.min.js"></script>

<sol-feed view="feed"
          source="https://rss.nytimes.com/services/xml/rss/nyt/World.xml"
          proxy="http://localhost:3002/proxy?uri="
></sol-feed>

AttributeRequiredDescription
view optional Layout: feed (default), topic, or all.
source required A feed URL (for feed) or <rdfFile>#<Topic> (for topic / all).
proxy optional A CORS proxy pattern prepended to cross-origin fetches.
default-selected optional (view="all" only) Comma- or pipe-separated list of feed labels or URL substrings to auto-check on the user's first visit (when nothing is in localStorage). Matching is case-insensitive substring containment against each feed's label OR URL, so "guardian" and "theguardian.com" both pick "The Guardian". Once the user touches the picker, their selection is persisted to localStorage and this attribute is ignored on subsequent visits.

view="feed"

Renders the one feed named by source as a list of article links. Clicking a link opens the article in the shared reader window.

<sol-feed view="feed"
  source="https://example.org/rss.xml"
  proxy="http://localhost:3002/proxy?uri="></sol-feed>

view="topic"

A single column scoped to one bk:Topic and its bk:subTopicOf subtree. The top pane shows ~5 feed sources at a time (scroll for more); below it, the article titles from the currently-selected source. The component fills its container's height and width — the demo here is 420px wide.

<sol-feed view="topic"
  source="data/feeds.ttl#News"
  proxy="http://localhost:3002/proxy?uri="></sol-feed>

view="all"

The top row holds one pill button per selected source (flush left) and the gear that reveals the source picker (flush right). Clicking a source button shows that feed's items in reverse chronological order, as a wrapping grid of image-on-left / title-on-right cards. Ticked sources are remembered in localStorage; on first visit the first source is auto-selected.

<sol-feed view="all"
  source="data/feeds.ttl#Feeds"
  proxy="http://localhost:3002/proxy?uri="></sol-feed>

source attribute

For view="feed" the source is the feed URL itself. For view="topic" and view="all" it is a URL of the form <rdfFile>#<Topic> — the fragment is required and names the bk:Topic the picker is scoped to. See the Listing feeds tab for the document shape.

proxy attribute

Most feeds are not served with permissive CORS headers, so a browser cannot fetch them directly. The proxy attribute is a pattern prepended to every URL the component fetches. If the pattern contains the literal token {url} it is replaced with the encoded URL; otherwise the encoded URL is appended.

Append style (the URL is added to the end): 
proxy="http://localhost:3002/proxy?uri="
Token style (the URL is substituted for {url}): 
proxy="https://proxy.example/fetch?u={url}"

Omit proxy to fetch directly — this works only for feeds that already send Access-Control-Allow-Origin.

Listing feeds for topic / all

The topic and all views read their feed list from an RDF/Turtle document. Two encodings are accepted, freely interchangeable — the W3C bookmark ontology (bk: / ui:) and SKOS (skos: / dct:). Point source at the document with a fragment selecting the topic the picker is scoped to:

view="topic" source="data/feeds.ttl#News"
view="all"   source="data/feeds.ttl#Feeds"

This path needs rdflib on the page (it publishes window.$rdf). The fragment is required — without it the component shows a clear error.

Bookmark form

Topics nest via bk:subTopicOf; each ui:Link is a feed and is tagged with a bk:hasTopic.

@prefix ui: <http://www.w3.org/ns/ui#> .
@prefix bk: <http://www.w3.org/2002/01/bookmark#> .

<#Feeds>   a bk:Topic ; ui:label "Feeds" .
<#News>    a bk:Topic ; ui:label "News" ;     bk:subTopicOf <#Feeds> .
<#SciTech> a bk:Topic ; ui:label "Sci/Tech" ; bk:subTopicOf <#Feeds> .

:00012 a ui:Link ;
  ui:label    "NY Times" ;
  bk:recalls  <https://rss.nytimes.com/services/xml/rss/nyt/World.xml> ;
  bk:hasTopic <#News> .

SKOS form

The root is a skos:ConceptScheme; its top concepts link back via skos:topConceptOf (or the inverse skos:hasTopConcept); the rest of the hierarchy uses skos:broader (or its inverse skos:narrower). Feeds are subjects identified by their URL, declared as rss:channel, labelled with dct:title, and tagged with dct:subject.

@prefix skos: <http://www.w3.org/2004/02/skos/core#> .
@prefix dct:  <http://purl.org/dc/terms/> .
@prefix rss:  <http://purl.org/rss/1.0/> .

<#Feeds>   a skos:ConceptScheme ; skos:prefLabel "Feeds" .
<#News>    a skos:Concept ; skos:prefLabel "News"     ; skos:topConceptOf <#Feeds> .
<#SciTech> a skos:Concept ; skos:prefLabel "Sci/Tech" ; skos:topConceptOf <#Feeds> .

<https://rss.nytimes.com/services/xml/rss/nyt/World.xml>
  a           rss:channel ;
  dct:title   "NY Times" ;
  dct:subject <#News> .

A SKOS twin of data/feeds.ttl lives at data/feeds-skos.ttl; the bookmark file also declares each bk:recalls URL as a rss:channel for consumers that want that typing directly.

Subtree scoping

The picker walks down from the focus topic in the fragment using every hierarchy predicate from either ontology, in either direction: bk:subTopicOf, skos:broader / skos:narrower, skos:topConceptOf / skos:hasTopConcept. So source="…#News" shows feeds tagged with News (and anything broader-than/narrower-than News), while source="…#Feeds" reaches every feed under that root.

Out-of-subtree links

Any feed whose topic is outside the focus subtree (or points at a URI that isn't declared as a topic) is silently dropped — only feeds whose topic is in the focus subtree appear in the picker.

Styling

sol-feed uses Shadow DOM, so its internals aren't reachable with a normal selector. Two ways in: shadow parts for elements the component exposes, and CSS custom properties that cascade through the shadow boundary.

Shadow parts (view="all")

PartElement
top-bar The source-pill buttons strip + gear toggle row above the articles grid.
articles The article-cards grid container itself (scrollable region).

Default look — and how to retint

Out of the box, view="all" paints a two-tone panel: a darker strip behind the top-bar (controls) and a lighter strip behind the articles grid, both relative to the page background via color-mix. The two strips sit flush and the outer corners are rounded so the pair reads as one continuous panel.

The simplest way to restyle is to set the two backdrop tokens on the host element (or globally on :root):

sol-feed {
  --feed-top-bar-bg:   #2c3e50;          /* solid dark control strip */
  --feed-articles-bg:  #ecf0f1;          /* solid pale article area */
}

For full control of the strips' padding / radius / border etc., drop down to ::part(top-bar) and ::part(articles):

sol-feed::part(top-bar)   { padding: 1rem; border-radius: 0; }
sol-feed::part(articles)  { padding: 2rem; border-radius: 0; }

To kill both backdrops entirely (flat, no panel look):

sol-feed {
  --feed-top-bar-bg:  transparent;
  --feed-articles-bg: transparent;
}
sol-feed::part(top-bar),
sol-feed::part(articles) { border-radius: 0; padding: 0; }

CSS custom properties

The component reads from the shared design tokens in web/styles/root.css. Override on :root (global) or on the host element (scoped):

TokenUsed for
--bg page background — inherited; the feed's outer host has no own bg.
--surface card backgrounds (article cards, picker fieldsets).
--border card borders, source-pill borders, list separators.
--text, --text-muted article body text and metadata.
--accent, --accent-dark selected pill background, focus rings, hover bg.
--link Article and source link colour — both .feed-link (link list) and .feed-card-title (image-card grid). Falls back to --accent when unset.
--hover, --focus-bg list-row hover and selected highlights.
--font-ui, --font-size typography on the host.
--feed-top-bar-bg backdrop behind the top-bar (controls strip). Default: color-mix(in srgb, var(--bg) 75%, #000).
--feed-articles-bg backdrop behind the articles grid. Default: color-mix(in srgb, var(--bg) 88%, #000).

Note: --divider-line

Earlier versions painted divider lines at the top of the host and between the top-bar and the articles grid using the project- wide --divider-line token. Those internal dividers were removed so the dashboard's two-tone tint (above) reads as one continuous panel. The --divider-line token still exists in root.css and is consumed by the dashboard's own header strip and other components that opt in.

Accessibility

The component is keyboard and screen-reader friendly:

  • Article and source links are real <a> elements and open in a new tab with rel="noopener noreferrer".
  • The news picker uses real <input type="checkbox"> controls inside a labelled <fieldset>.
  • The source list in view="topic" is a navigation landmark; the selected source carries aria-current.
  • Loading and error messages are announced through a polite aria-live region.
  • News-card descriptions reveal on keyboard focus as well as hover, so the overlay text is never mouse-only.
  • All colours come from the shared theme tokens, so the component follows the page's light / dark setting.
