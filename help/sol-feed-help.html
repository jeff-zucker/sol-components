sol-feed component

Fetches and displays RSS or Atom feeds. One element, three layouts — pick a layout with the view attribute: a single feed, a multiple-source picker, or a news-page grid that merges and randomizes articles from the sources you tick. Cross-origin feeds are fetched through an optional CORS proxy; both RSS and Atom are parsed, and the parser pulls out titles, links, dates, descriptions and a representative image per article.

AttributeRequiredDescription
view optional Layout: single (default), multiple, or news-page.
source see note A feed URL (for single) or a resource that lists feeds (for multiple / news-page).
proxy optional A CORS proxy pattern prepended to every fetched URL.

view="single"

Renders the one feed named by source as a list of article links. Each link opens the article in a new tab and shows the feed name and publication date beneath the title.

<sol-feed view="single"
  source="https://example.org/rss.xml"
  proxy="http://localhost:3002/proxy?uri="></sol-feed>

view="multiple"

A two-pane sources picker. A side list shows every feed; selecting one fetches it and lists its articles in the content pane. The first feed is selected automatically.

<sol-feed view="multiple"
  source="https://example.org/feeds.ttl"
  proxy="http://localhost:3002/proxy?uri="></sol-feed>

view="news-page"

A Google-News-like grid. Each feed gets a checkbox; articles from every ticked source are merged, shuffled into a random order, and shown as image cards. Each card shows the article image with its title; the description is revealed as an overlay on hover or keyboard focus. Toggling a checkbox updates the grid live — no extra button to press.

<sol-feed view="news-page" proxy="http://localhost:3002/proxy?uri=">
  <a href="https://a.example/rss">Site A</a>
  <a href="https://b.example/atom">Site B</a>
</sol-feed>

source attribute

For view="single" the source is the feed URL itself. For view="multiple" and view="news-page" it is instead a resource that lists feeds — see the Listing feeds tab. In those two views source may be omitted and the feed list taken from inline <a> children instead.

proxy attribute

Most feeds are not served with permissive CORS headers, so a browser cannot fetch them directly. The proxy attribute is a pattern prepended to every URL the component fetches. If the pattern contains the literal token {url} it is replaced with the encoded URL; otherwise the encoded URL is appended.

Append style (the URL is added to the end): 
proxy="http://localhost:3002/proxy?uri="
Token style (the URL is substituted for {url}): 
proxy="https://proxy.example/fetch?u={url}"

Omit proxy to fetch directly — this works only for feeds that already send Access-Control-Allow-Origin.

Listing feeds for multiple / news-page

The multiple and news-page views need a list of feeds. There are three ways to provide one.

  • Inline anchors — drop <a href> children inside the element. The link text becomes the feed's label. This needs no source attribute at all. 
    <sol-feed view="news-page" proxy="http://localhost:3002/proxy?uri=">
  <a href="https://feeds.bbci.co.uk/news/rss.xml">BBC News</a>
  <a href="https://feeds.npr.org/1001/rss.xml">NPR</a>
</sol-feed>
  • An HTML page — point source at an HTML document; every <a href> in it becomes a feed.
  • An RDF / Turtle bookmark file — point source at a Turtle document using the W3C bookmark ontology, as in data/feeds.ttl. Each ui:Link becomes a feed: ui:label is its name, bk:recalls the feed URL, and bk:hasTopic groups it under a bk:Topic. The picker and source list are organised by those topics. This path needs rdflib present on the page (it publishes window.$rdf). 
    @prefix ui: <http://www.w3.org/ns/ui#> .
@prefix bk: <http://www.w3.org/2002/01/bookmark#> .

<#News> a bk:Topic ; ui:label "News" .

:00012 a ui:Link ;
  ui:label    "NY Times" ;
  bk:recalls  <https://rss.nytimes.com/services/xml/rss/nyt/World.xml> ;
  bk:hasTopic <#News> .

Accessibility

The component is keyboard and screen-reader friendly:

  • Article and source links are real <a> elements and open in a new tab with rel="noopener noreferrer".
  • The news picker uses real <input type="checkbox"> controls inside a labelled <fieldset>.
  • The source list in view="multiple" is a navigation landmark; the selected source carries aria-current.
  • Loading and error messages are announced through a polite aria-live region.
  • News-card descriptions reveal on keyboard focus as well as hover, so the overlay text is never mouse-only.
  • All colours come from the shared theme tokens, so the component follows the page's light / dark setting.
